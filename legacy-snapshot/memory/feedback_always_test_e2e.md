SEMPRE testar de ponta a ponta antes de dizer que uma tarefa está concluída.
Nunca entregar código sem verificar que funciona no ambiente real.

**Why:** Nelson perdeu confiança quando entreguei código com erro 2x seguidas. Testar antes evita retrabalho e frustração.
**How to apply:** Após qualquer mudança: instalar, rodar, verificar logs, confirmar que funciona. Só então reportar como concluído.
